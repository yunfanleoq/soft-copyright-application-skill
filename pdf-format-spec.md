# 鉴别材料 PDF 格式规范（版权中心实务要求）

> **法规底线**：《计算机软件著作权登记办法》第十条（前/后各 30 页、程序 ≥50 行/页、文档 ≥30 行/页）。  
> **实务细则**：各地软协/代办机构转述的中国版权保护中心（CPCC）线上填报要求，与中心当期系统提示冲突时 **以系统为准**。  
> 官方入口：https://www.ccopyright.com.cn/

---

## 一、通用要求（源程序 + 文档均适用）

| 项目 | 要求 |
|------|------|
| 文件格式 | **PDF**（线上登记鉴别材料须上传 PDF；勿传仅 Word） |
| 纸张 | A4（210mm×297mm），**纵向** |
| 打印/导出 | **单面**（禁止双面，便于审查装订） |
| 颜色 | 鉴别材料 **黑白**（截图手册亦建议灰度或彩色打印后转清晰 PDF，以当地要求为准） |
| 加密与水印 | **不得** PDF 加密；**不得**第三方工具水印（会导致系统无法识别） |
| 文件大小 | 单文件通常 ≤ **50MB**（超限拆分并标注 1/2、2/2） |
| 分辨率 | 扫描件建议 ≥ **300 DPI**；电子版导出以文字可选中、清晰为准 |
| 页眉 | **软件全称 + 版本号**（与申请表完全一致，含是否带 `V`） |
| 页码位置 | **右上角**连续阿拉伯数字 |
| 页脚 | **著作权人全称**（与申请表一致） |
| 名称一致性 | 页眉、页脚、正文、**界面截图内**出现的软件名、版本号、著作权人均须一致 |

### 页码编排（常见实务）

| 材料 | 页码规则 |
|------|----------|
| 源程序 PDF | 交存的 60 页（前 30+后 30）整体编 **第 1 页～第 60 页**；封面/说明页一般不计入 |
| 文档 PDF | 自 **目录或正文第 1 页** 起连续编至第 60 页（前 30+后 30）；封面可不显示页码 |
| 分册上传 | 若系统要求一个 PDF，则将前 30 与后 30 **合并为一个文件** 连续编页码 |

## 二、源程序鉴别材料 PDF

### 2.1 交存页数

| 代码规模 | 交存方式 |
|----------|----------|
| 折算 ≥ 60 页（50 行/页） | **前连续 30 页 + 后连续 30 页**，共 60 页 |
| 不足 60 页 | **全部**源程序 |
| 实务参考 | 有机构提示「超过约 3000 行按 60 页交存」——以代码页数核算为准 |

### 2.2 每页行数

| 规则 | 说明 |
|------|------|
| 常规页 | **不少于 50 行有效代码**（空行不计） |
| 结束页 | 最后一页可例外，但实务要求 **不少于 15 行** |
| 字体 | **等宽字体**（Courier New、Consolas 等），建议 **9～10.5 磅** |
| 换行 | 保持缩进；避免长行被截断导致语法不可读 |

### 2.3 页眉页脚示例

```text
页眉（左）    知识产权智能体平台 V1.0                    页眉（右）  第 12 页 共 60 页
页脚（中）                    XX科技有限公司
```

文档类型标注可写作「源程序」或「源代码」（与代办机构口径一致即可）。

### 2.4 内容禁忌（审查常查）

- 删除或统一替换：与申请表不符的 `author`、`copyright`、`version`、`公司名称`
- 不得出现 **开发完成日期之后** 的版权年份（除非与事实一致）
- 脱敏：API Key、密码、内网 IP、客户真实数据
- 避免整页无意义注释、LICENSE 堆砌充行数

### 2.5 推荐 PDF 产出方式

1. **自动**：`scripts/generate_source_pdf.py`（见 SKILL Step 2b）  
2. **半自动**：`02/03` txt → Word 套用页眉页脚 → 导出 PDF  
3. **提交前**：用 Adobe/浏览器打印预览核对每页行数、页码连续、中文不乱码

## 三、文档鉴别材料 PDF（用户操作手册）

### 3.1 文档类型（任选一种）

| 类型 | 适用 | CPCC 常见要求侧重 |
|------|------|-------------------|
| **用户操作手册** | 面向最终用户 | **登录界面**、分步操作、**界面截图**、文字说明 |
| 设计说明书 | 技术审查 | 结构图、模块图、流程图、接口与算法说明 |
| 使用说明书 | 通用 | 功能说明 + 使用方法 |

**本 Skill 默认采用「用户操作手册」**，须 **实机运行截图**，不可用设计稿/线框图代替。

### 3.2 交存页数与行数

| 规则 | 说明 |
|------|------|
| 页数 | 前 **30** 页 + 后 **30** 页；不足 60 页交 **全部** |
| 行数 | 除附图页外，每页 **不少于 30 行**（含正文、图题、步骤说明） |
| 附图页 | 含大图时可少于 30 行，但不宜连续多页无文字说明 |

### 3.3 内容与截图要求（核心）

必须基于 **本软件真实运行环境**：

1. **启动系统**（本地或测试环境），使用与说明书一致的版本  
2. 按功能模块 **逐步操作**，每步包含：  
   - **步骤编号**（步骤 1、步骤 2…）  
   - **文字说明**（点击哪里、输入什么、预期结果）  
   - **界面截图**（整屏或关键区域，分辨率建议 ≥ 1280 宽）  
3. **登录界面**：须有单独小节 + 截图（含软件名称/版本可见更佳）  
4. 截图内窗口标题、关于页、登录页中的 **软件名、版本号、单位名** 与申请表一致  
5. 必要时对截图关键按钮 **红框/序号标注**  
6. 可附 **功能结构图/业务流程图**（Visio、draw.io、Mermaid 导出）

### 3.4 页眉页脚示例

```text
页眉（左）    知识产权智能体平台 V1.0  操作手册        页眉（右）  第 8 页 共 60 页
页脚（中）                    XX科技有限公司
```

### 3.5 手册章节最低配置（用于凑足 60 页且过审）

| 章节 | 建议页数 | 截图数量（参考） |
|------|----------|------------------|
| 封面、目录 | 2～4 | 0 |
| 引言、运行环境 | 4～6 | 1（架构图） |
| 安装部署 | 4～8 | 2～4（终端/浏览器） |
| 登录与首页 | 4～6 | 3～5 |
| 核心业务模块 × N | 每模块 6～12 | 每模块 ≥ 4 |
| 系统设置、FAQ | 4～8 | 2～4 |

### 3.6 推荐产出流程

见 [manual-screenshot-workflow.md](manual-screenshot-workflow.md) 与 `scripts/generate_manual_pdf.py`（合并截图版 Word/PDF 需人工排版时，Agent 生成 `screenshot-manifest.md` 与分步文稿）。

## 四、线上上传文件命名（建议）

```text
{著作权人全称}-{软件全称}{版本号}-源程序.pdf
{著作权人全称}-{软件全称}{版本号}-操作手册.pdf
```

示例：`某某科技有限公司-知识产权智能体平台V1.0-源程序.pdf`

## 五、提交前 PDF 自检清单

| □ | 检查项 |
|---|--------|
| □ | 文件为 PDF，可正常打开，无密码 |
| □ | 源程序共 60 页（或全部页），页码 1～N 连续 |
| □ | 源程序每页（除末页）≥ 50 行有效代码 |
| □ | 操作手册共 60 页（或全部页），含登录与主要功能截图 |
| □ | 页眉软件名、版本号与申请表一致 |
| □ | 页脚著作权人全称与申请表一致 |
| □ | 截图来自本软件运行界面，非设计稿 |
| □ | 单文件 < 50MB |
| □ | 打印预览：A4、单面、文字清晰 |
