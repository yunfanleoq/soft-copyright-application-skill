- 中国版权保护中心：https://www.ccopyright.com.cn/
- 《计算机软件著作权登记办法》：国家版权局令第 1 号（现行有效条文以官网 PDF 为准）

## 附录：常用软件分类号（节选）

> 填报时在系统中选择最接近项；下列为常见企业软件分类，**以中心系统下拉选项为准**。

| 分类号 | 名称 | 适用场景 |
|--------|------|----------|
| 30200-0000 | 行业应用软件 | 面向特定行业（知识产权、医药等） |
| 30100-0000 | 通用应用软件 | 通用办公、管理类 |
| 30300-0000 | 嵌入式软件 | 嵌入硬件设备 |
| 30400-0000 | 中间件 | 数据库中间件、消息中间件等 |
| 30500-0000 | 操作系统 | 系统软件 |

**IP 代理/智能体类** 通常选 **行业应用软件** 或 **通用应用软件**，按是否面向单一行业在说明书中论证。

## 源程序交存顺序建议

1. 应用入口（`main.py`、`main.ts`）
2. 核心业务（流水线、解析器、Agent 编排）
3. API 路由层
4. 领域模型与服务
5. 前端页面与状态管理（体现 UI 逻辑部分）
6. 避免：纯数据种子、迁移脚本堆砌、自动生成的 lock 文件

## 行数统计口径

| 口径 | 说明 |
|------|------|
| 法规底线 | 程序每页 **≥50 行**（登记办法第十条） |
| 本 Skill 脚本默认 | 非空行计入有效行；空行不计入页内 50 行填充 |
| 注释 | 法规未明确排除注释；实务中可保留必要注释，避免整页注释 |

仅当涉及商业秘密且无法全文交存时选用；遮盖部分不得超过交存源程序的 50%。

## 常见问题

### Q1：开源组件要不要交存？
交存的是**本软件源程序**；若大量引用第三方库，应在说明书中说明，并备许可协议；交存代码应体现自研逻辑。

### Q2：前后端分两个软著吗？
可作为一套「软件」统一登记（前后端同属一个产品），也可按产品规划拆分；需与用户确认产品边界。

### Q3：版本 V1.0 与 1.0？
证书与材料以申请表填报为准；页眉版本号须与申请表一致（含是否带 `V`）。

### Q4：数据库脚本算不算源程序？
业务相关的 SQL/迁移可少量纳入；大量 seed 数据不宜作为主要交存内容。

### Q5：鉴别材料是否必须 PDF？
**是。** 中国版权保护中心线上填报鉴别材料须 **PDF** 上传。本 Skill 提供 `generate_source_pdf.py` 自动生成源程序 PDF；操作手册建议 Word 排版（页眉右上页码、页脚著作权人）后导出 PDF。

### Q6：源程序 PDF 页眉页脚怎么写？
- 页眉左：`{软件全称} {版本号} 源程序`
- 页眉右：`第 n 页 共 60 页`（右上角）
- 页脚中：著作权人全称  
详见 [pdf-format-spec.md](pdf-format-spec.md)。

### Q7：操作手册一定要截图吗？
**用户操作手册**须图文结合：含 **登录界面**、关键步骤 **实机截图** 与文字说明；截图内名称、版本须与申请表一致。不能仅用文字堆砌或设计稿代替。
